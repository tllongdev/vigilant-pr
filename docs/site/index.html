Vigilant PR is a portable, workflow-agnostic pull-request reviewer. It goes to the repo, reviews the PR, and posts severity-tagged inline comments on behalf of you - your GitHub identity, not a generic bot. No per-repo setup.

Get started View on GitHub

What it does

The review engine, proven in production, repackaged so it needs no GitHub Actions install.

Two-tier reasoning

Fast Sonnet by default; escalate to Opus with adaptive thinking for the hard PRs where subtle bugs hide.

Adversarial by design

Assumes the code is wrong until proven correct. Every finding cites a concrete path:line and a real failure mode.

Thread-aware re-review

Reads its own prior comments and your replies, validates or re-flags them, and scopes to the incremental diff.

No noise ratchet

Dedups already-raised findings, caps nits, and stays silent on a re-push when there is nothing new to say.

Posts as one review

The whole review lands in a single GitHub call - one notification, not N. Never auto-approves or blocks.

Honest signature

Every comment states it is an AI-assisted first pass commissioned by you, and names the model that ran.

Run any model

Model-agnostic: Claude for the deepest reviews, or free no-card tiers (Groq, Gemini, NVIDIA), or a local model. One provider/model string.

Critical - must fix before merge Medium - real issue, fix before merge Nit - minor, non-blocking

Quickstart

You need the GitHub CLI authenticated as you (or a GH_TOKEN) and a model key - Claude, or a free tier like Groq/Gemini/NVIDIA.

# Install (no-Docker path; needs `gh` on PATH)
pipx install git+https://github.com/tllongdev/vigilant-pr

# Run free: grab a no-card Groq key at console.groq.com/keys
export GROQ_API_KEY=gsk_...
export VIGILANT_MODEL=groq/llama-3.3-70b-versatile

# Review a PR and post as you
vigilant review https://github.com/owner/repo/pull/123

# Or bring Claude for the deepest reviews
export ANTHROPIC_API_KEY=sk-ant-...
vigilant review 123 --repo owner/repo --opus

# Preview without posting  (or VIGILANT_MODEL=mock for a keyless demo)
vigilant review 123 --repo owner/repo --dry-run

Run the watcher as a daemon

Polling only - no inbound ports. Auto-reviews PRs where you are tagged.

docker run -d --name vigilant-pr --restart unless-stopped \
  -e ANTHROPIC_API_KEY -e GH_TOKEN \
  -e VIGILANT_ORG_ALLOW="acme" \
  -v vigilant-state:/root/.vigilant \
  -e VIGILANT_SEEN_PATH=/root/.vigilant/seen.json \
  ghcr.io/tllongdev/vigilant-pr:latest watch
Your review, your identity. Comments are authored by your GitHub token, so they are your review - carrying an honest AI-assisted signature. Vigilant PR never uses APPROVE or REQUEST_CHANGES; a human still owns the merge decision.
